Reflexión sobre el Desarrollo del Portafolio

Defensa del Proyecto

Este portafolio fue desarrollado con el objetivo de mostrar mis habilidades en desarrollo web y facilitar el contacto con potenciales empleadores. Se diseñó para ser visualmente atractivo, profesional y fácil de navegar, con un enfoque en la presentación clara de mi perfil, experiencia y proyectos.

El diseño del portafolio sigue principios de UX/UI modernos, asegurando una estructura bien organizada con secciones definidas: presentación, habilidades, proyectos y contacto. Se utilizó Bootstrap para el diseño responsivo, garantizando una visualización óptima en diferentes dispositivos.

Ejemplo de Código

A continuación, un fragmento de código de la estructura básica de la sección principal:

                    <section class="container text-center my-5">
            <h1 class="display-4">¡Hola! Soy Sara Chbali</h1>
            <p class="lead">Desarrolladora web apasionada por la tecnología y la innovación.</p>
            <a href="#contacto" class="btn btn-success btn-lg">Contactar</a>
        </section>

Aquí se utiliza display-4 para el título, lead para el subtítulo y btn btn-success btn-lg para estilizar el botón con Bootstrap.

Tecnologías Utilizadas

El portafolio fue desarrollado utilizando herramientas modernas para garantizar su rendimiento y compatibilidad.

Bootstrap 5

Framework CSS utilizado para la maquetación y estilos responsivos sin necesidad de escribir mucho CSS personalizado.

JavaScript

Se empleó para agregar interactividad, animaciones y efectos al desplazarse por la página.

HTML y CSS

Se utilizó HTML semántico para mejorar la accesibilidad y una pequeña cantidad de CSS para personalizar detalles.

Ejemplo de Código

Aprendizaje y Problemas Encontrados

Durante el desarrollo, enfrenté varios desafíos que me permitieron aprender y mejorar mis habilidades.

Adaptabilidad del diseño

En un principio, algunos elementos no se adaptaban correctamente a pantallas pequeñas. Para solucionar esto, se usaron clases responsivas de Bootstrap como col-md-6 y col-lg-4 para ajustar el diseño según el tamaño del dispositivo. 

        <div class="row">
            <div class="col-md-6 col-lg-4">
                <div class="card">
                    <div class="card-body">
                        <h5 class="card-title">Proyecto 1</h5>
                        <p class="card-text">Descripción.</p>
                    </div>
                </div>
            </div>
        </div>

Botón de descarga de CV no funcionaba correctamente

Inicialmente, el botón de descarga de mi CV en PDF no hacía la acción esperada. Se solucionó asegurando que el archivo estuviera correctamente vinculado y usando la propiedad download. 

        <a href="CV_SaraChbali.pdf" download class="btn btn-outline-success">Descargar CV</a>

Animaciones de entrada al hacer scroll

Se implementó un efecto de entrada para que los elementos aparecieran suavemente al desplazarse por la página, usando JavaScript. 

        // JavaScript para activar la animación al hacer scroll
        document.addEventListener("DOMContentLoaded", function () {
            let elements = document.querySelectorAll(".fade-in");
        
            function checkScroll() {
                elements.forEach(el => {
                    if (el.getBoundingClientRect().top < window.innerHeight * 0.85) {
                        el.classList.add("show");
                    }
                });
            }
        
            window.addEventListener("scroll", checkScroll);
        });

En CSS se agregó: 

        /* Estilos CSS para animaciones de entrada */
        .fade-in {
            opacity: 0;
            transform: translateY(20px);
            transition: opacity 0.6s ease, transform 0.6s ease;
        }
        
        .fade-in.show {
            opacity: 1;
            transform: translateY(0);
        }

Accesibilidad y Usabilidad

Se tomaron medidas para garantizar que el portafolio sea accesible para todos los usuarios.

Uso de etiquetas semánticas

Se implementaron <header>, <nav>, <main> y <footer> para mejorar la estructura del sitio y facilitar la navegación a usuarios con tecnologías de asistencia.

Contraste adecuado

Se eligieron colores con buen contraste para facilitar la lectura y cumplir con los estándares de accesibilidad WCAG. Se evitó el uso de combinaciones de colores que dificulten la visibilidad a personas con daltonismo.

Navegación intuitiva

Se usaron aria-labels y tabindex para mejorar la experiencia de usuarios con lectores de pantalla y permitir una navegación fluida mediante el teclado.

        <button class="btn btn-success" aria-label="Contactar">Contactar</button>

Conclusiones y Futuras Mejoras

El portafolio ha sido un gran ejercicio de aprendizaje y ha cumplido con su propósito de mostrar mis habilidades. Sin embargo, hay aspectos que pueden mejorar en el futuro.

Optimización para SEO

Se pueden mejorar las meta etiquetas y la estructura del sitio para aumentar la visibilidad en buscadores y mejorar el posicionamiento en Google.

Implementación de un modo oscuro

Agregar un interruptor para alternar entre modo claro y oscuro mejoraría la experiencia del usuario, permitiendo una navegación más cómoda en diferentes condiciones de luz. 

        <button onclick="toggleDarkMode()" class="btn btn-dark">Modo Oscuro</button>
                    
        function toggleDarkMode() {
            document.body.classList.toggle("dark-mode");
        }
                    
        .dark-mode {
            background-color: #121212;
            color: #ffffff;
        }
